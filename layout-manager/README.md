# Layout Manager Pro

**Автоматический переключатель раскладки клавиатуры для Windows**

Программа автоматически меняет раскладку клавиатуры в зависимости от активного окна (программы).

---

## 🎯 Что Это Делает?

| Без Программы | С Программой |
|---------------|--------------|
| ❌ Забыл переключить раскладку → UI сломался | ✅ Само переключает при открытии программы |
| ❌ COVAS не понимает русские команды | ✅ Для COVAS всегда Russian |
| ❌ EDCoPilot требует English | ✅ Для EDCoPilot всегда English |
| ❌ Постоянно переключаешь вручную | ✅ Работает в фоне, ничего не делаешь |

---

## 🚀 Быстрый Старт

### Шаг 1: Запуск

```bash
python main.py
```

### Шаг 2: Добавление Процессов

1. Введите имя процесса (например, `EliteDangerous64.exe`)
2. Выберите язык (English или Russian)
3. Нажмите **"Добавить"**

### Шаг 3: Сохранение

Нажмите **"Сохранить"** для записи в `config.ini`

### Шаг 4: Запуск Монитора

Нажмите **"Запустить"** для запуска фонового монитора раскладки

При запуске автоматически закрываются предыдущие экземпляры программы.

### Шаг 5: Выход

Нажмите **"Выход"** для полного завершения приложения

## ⚙️ Формат config.ini

```ini
[EliteDangerous64.exe]
Language=English
LayoutID=0x4090409

[COVAS NEXT.exe]
Language=Russian
LayoutID=0x4190419

[EDCoPilotGUI2.exe]
Language=English
LayoutID=0x4090409
```

### Поля

| Поле | Описание | Пример |
|------|----------|--------|
| `[ProcessName.exe]` | Имя процесса | `EliteDangerous64.exe` |
| `Language` | Человекочитаемое название | `English`, `Russian` |
| `LayoutID` | Код раскладки Windows (hex) | `0x4090409` |

### LayoutID Раскладок

| Раскладка | LayoutID | Hex |
|-----------|----------|-----|
| **English US** | 1033 | `0x4090409` |
| **Russian** | 1049 | `0x4190419` |

|------------|----------|

| **Ctrl+Alt+X** | Выход из AHK скрипта |

## 🔧 Как Это Работает

### Python GUI (main.py)

1. Создаёт окно 450x450 пикселей
2. Позволяет добавлять/удалять процессы
3. Сохраняет в `config.ini` через `configparser`
4. Запускает `monitor.ahk` через `subprocess`

### AutoHotKey (monitor.ahk)

1. Читает `config.ini` при запуске
2. Каждые **500мс** проверяет активное окно
3. При совпадении отправляет **PostMessage 0x50** (WM_INPUTLANGCHANGEREQUEST)
4. Windows меняет раскладку

### Windows API

```
PostMessage 0x50 (WM_INPUTLANGCHANGEREQUEST)
- wParam = 0 (не важно)
- lParam = LayoutID (например, 0x4090409)
```

## ❓ Troubleshooting

### Проблема: AutoHotKey не найден

**Решение:**
1. Установите AutoHotKey v1.1
2. Добавьте в PATH или укажите полный путь в настройках

### Проблема: Раскладка не переключается

**Решение:**
1. Проверьте что `monitor.ahk` запущен (иконка в трее)
2. Проверьте имя процесса в `config.ini` (должно совпадать точно)
3. Перезапустите AHK скрипт через Ctrl+Alt+X и заново через GUI

### Проблема: Неверное имя процесса

**Решение:**
1. Откройте Диспетчер Задач
2. Найдите процесс в вкладке "Подробности"
3. Скопируйте точное имя (например, `EliteDangerous64.exe`)

### Проблема: GUI не запускается

**Решение:**
1. Проверьте версию Python: `python --version`
2. Убедитесь что tkinter установлен (встроен в Python 3.8+)

## 📊 Сравнение Компонентов

| Компонент | Язык | Назначение |
|-----------|------|------------|
| **main.py** | Python | GUI для управления настройками |

### Почему Такая Архитектура?

| Решение | Причина |
|---------|---------|
| **Python для GUI** | tkinter встроен, не нужно ставить зависимости |
| **AHK для фона** | Идеально подходит для переключения раскладки |
| **INI файл** | Простой формат, читается без библиотек |
| **Разделение** | Каждый компонент делает одну работу хорошо |
